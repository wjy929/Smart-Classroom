## Git 协作规范

### 分支策略
- `main`: 主分支，保护分支，存放稳定版本。
- `dev`: 开发分支，用于集成功能。
- `feature/xxx`: 功能分支，个人在此分支工作。

### 提交信息规范
格式：`<类型>: <描述>`
示例：
- `feat: 完成角色分析与用例列表`
- `docs: 更新需求规格说明书1.1章节`
- `fix: 修正活动图逻辑错误`

### 工作流程
1. 从 `dev` 分支创建 `feature/xxx` 分支。
2. 在功能分支上工作并提交。
3. 完成后，向 `dev` 分支发起 Pull Request。
4. 经代码审查后合并。
